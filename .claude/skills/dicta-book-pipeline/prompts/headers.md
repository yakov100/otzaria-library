# Header Identification Prompt — Pass A

> זה ה‑prompt שאתה (Claude) מחיל על עצמך כשאתה ב‑Stage 4 של ה‑pipeline.

## המטרה

לעבור על קובץ TXT של ספר דיקטה אחרי ניקוי דטרמיניסטי (5.2), ולהציע כותרות `<h2>`–`<h6>` שלא קיימות בו עדיין. שם הספר (`<h1>`) ושם המחבר (שורה 2) **כבר** מקודדים — אסור לגעת בהם.

## כללי קודש

1. **אסור להמציא תוכן.** אסור להוסיף ולו אות אחת שלא קיימת בקובץ. רק עוטפים תגי כותרת על טקסט שכבר שם.
2. **אסור לערוך את הקובץ.** הפלט הוא JSON של הצעות בלבד.
3. **שמור על היררכיה לוגית.** רמה N לא תופיע ישר בתוך רמה N‑2 ללא רמה N‑1 מתווכת.
4. **חוסר ודאות → confidence נמוך.** עדיף להציע שגוי בביטחון נמוך מאשר לפסוח על הצעה. המשתמש יסקור את ה‑low.
5. **שמור על תגים קיימים** (`<b>`, `<big>`, `<small>`). אם המילה שאתה רוצה להפוך לכותרת כבר עטופה ב‑`<b>` — הצעה תהיה: להחליף `<b>X</b>` ב‑`<h?>X</h?>`.

## כללי טיפוס לפי ז'אנר

עוצמת ההיררכיה משתנה לפי סוג הספר. זהה תחילה את הסוג:

### תלמוד / פירושי תלמוד (גמרא, ראשונים, אחרונים)
- `<h2>` — מסכת ("מסכת ברכות") או פרק ("פרק ראשון - מבוי")
- `<h3>` — "דף ב.", "דף ב:" (כל עמוד נפרד)
- `<h4>`–`<h5>` — לרוב לא נחוץ
- **רמות יחסיות**: אם יש כותרת מסכת/פרק מעל הדפים → דף=`<h3>`. אם הדף הוא היחידה העליונה (אין פרק/מסכת) → דף=`<h2>`, והפרק (אם מופיע) נשאר שורת body מודגשת `<big><b>פרק שני</b></big>`, לא כותרת.
- **שם כותרת הפרק קנוני** — בפורמט `<שם הפרק> - פרק <מספר>`. ראה "נורמול טקסט הכותרת" למטה.
- **`מתני'`/`גמ'` אינם כותרות** — הם תחילת למה. עטוף inline: `<big><b>מתני':</b></big>`, `<big><b>גמרא:</b></big>` (הרחב `גמ'`→`גמרא`, הוסף `:`).
- ראה דוגמה: [גאון יעקב על עירובין](../../../../DictaToOtzaria/ערוך/ספרים/אוצריא/תלמוד%20בבלי/אחרונים/גאון%20יעקב%20על%20עירובין.txt)

### שולחן ערוך / פוסקים
- `<h2>` — חלקים (אורח חיים, יורה דעה, וכד')
- `<h3>` — סימן (סימן א, סימן ב, ...)
- `<h4>` — סעיף

### חסידות / מוסר / דרוש
- `<h2>` — מסכת/מאמר ראשי, "הקדמה", "מפתחות"
- `<h3>` — פרשה / פרק
- `<h4>` — קטע ספציפי אם יש מספור
- ראה דוגמה: [אמרי צדיקים](../../../../DictaToOtzaria/ערוך/ספרים/אוצריא/חסידות/ספרי%20חסידות%20נוספים/אמרי%20צדיקים.txt)

### תנ"ך + פירוש
- `<h2>` — חומש / ספר
- `<h3>` — פרשה / פרק
- `<h4>` — פסוק

### קבלה
- היררכיה מגוונת. הזהר. אם לא ברור — סמן confidence: low.

## דפוסי זיהוי שכיחים

| תבנית בטקסט | הצעה |
|---|---|
| שורה שכל תוכנה `<b>פרק ראשון</b>` או `<b>פרק</b> <b>ראשון</b>` | `<h2>פרק ראשון</h2>` |
| `<b>דף ב'</b>` או `<b>דף</b> <b>ב'</b>` | `<h3>דף ב.</h3>` (נורמליזציה לסגנון אוצריא) |
| `<b>סימן א</b>` בתחילת שורה | `<h3>סימן א</h3>` או `<h4>` לפי הקשר |
| `<b>הקדמה</b>` בלבד בשורה | `<h2>הקדמה</h2>` |
| `<big>X</big>` כתואר/כותרת בתוך שורת body | **סלקטיבי** — בודד כ‑h2/h3 רק אם זו כותרת מבנית (פרשה/חלק/מסכת). `<big>` להדגשה בתוך משפט נשאר במקומו. אל תניח "כל `<big>` = כותרת". |
| `<b>ע"א</b>` או `<b>ע"ב</b>` בלבד | אל תהפוך לכותרת — זו כותרת חלקית. הצע למזג עם הדף הקרוב |
| מספר משלי/סעיף inline בראש קטע `(א)` | בודד ל‑`<h4>(א)</h4>` ושים את הטקסט בשורה הבאה |

## נורמול טקסט הכותרת

הצעת כותרת אינה רק עטיפת התגים — גם **תוכן** הכותרת נכתב בצורה הקנונית של אוצריא, לא כפי שהודפס. החל את ארבעת הכללים:

1. **הסר שם מחבר** — שם המחבר כבר ב‑`<h1>`, אסור לחזור עליו בכותרת פנימית.
   `חדושי פרקא קמא דכתובות מהרי"ט` → `חדושי פרקא קמא דכתובות`
   `פרקא קמא דשבת מהרי"ט` → `פרקא קמא דשבת`
2. **פתח קיצורים** — `פ'` → `פרק`, וכד'.
   `פ' האומר בקונטרס` → `פרק האומר בקונטרס`
3. **כותרת פרק בפורמט `<שם הפרק> - פרק <מספר>`** — שם הפרק המקובל בש"ס תחילה, מקף, ואז המילה "פרק" + המספר. כשבמקור מופיע מספר חשוף בלבד — הוסף את שם הפרק הקנוני מהמסכת.
   `שלהי פרק המדיר` → `שלהי פרק המדיר - פרק שביעי`
   `פרק ח'` (כתובות) → `האשה שנפלו - פרק שמיני`
   `מי שהיה נשוי פרק י'` → `מי שהיה נשוי - פרק עשירי`
   `פרק שני דייני גזרות` → `שני דייני גזרות - פרק יג`
4. **מספור הפרק** — אורדינל 1–10 במילים (ראשון, שני … עשירי); 11 ומעלה בגימטריה (יא, יב, יג).

> כלל 3 דורש ידע בשמות פרקי הש"ס המקובלים (כתובות פרק ח = "האשה שנפלו"). זו הצעת `confidence: medium` שדורשת אישור אדם — ציין במפורש ב‑`note` מהו שם הפרק שהוספת ומניין.

## מה להציג למשתמש בסיום

- מספר הצעות לפי רמה: h2 / h3 / h4 / h5 / h6
- מספר הצעות לפי confidence: high / medium / low
- 3 דוגמאות מייצגות (אחת מכל confidence)
- שאלה: להחיל אוטומטית רק את ה‑high, או לסקור הכל?

## אם אתה מזהה בעיית מבנה

לדוגמה — h3 שמופיע ישר אחרי h1 ללא h2:

```json
{
  "line": 0,
  "type": "structure_warning",
  "message": "אין h2 בכל הספר. ייתכן שצריך להוסיף h2 'הקדמה' לפני שורה X.",
  "confidence": "medium"
}
```

הפק entry כזה ב‑JSON; אבל אל תקבע באופן עצמאי שזה מחייב הוספה. הצג למשתמש.
